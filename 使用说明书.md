# VitalPet（活力宠物）使用说明书

VitalPet 是一个运动驱动的电子宠物 Web App。用户通过 iPhone / Apple Watch 的 Apple Health 步数获得猫咪口粮，再在网页中喂养像素风白猫，提升宠物等级。

## 1. 项目结构

```text
Vispet/
├── backend/
│   ├── app.py              # Flask 后端 API 与静态页面服务
│   └── vitalpet.db         # SQLite 数据库，首次运行后自动生成或初始化
├── frontend/
│   ├── index.html          # Web 页面
│   ├── style.css           # 像素风样式
│   ├── script.js           # 前端交互逻辑
│   └── catcat.png          # 白猫图片
├── requirements.txt        # Python 依赖
├── Procfile                # Render / Gunicorn 启动命令
└── 使用说明书.md
```

## 2. 本地启动

进入项目目录：

```bash
cd C:\finalproject\Vispet
```

安装依赖：

```bash
pip install -r requirements.txt
```

启动 Flask 服务：

```bash
python backend\app.py
```

打开浏览器访问：

```text
http://127.0.0.1:5000/
```

## 3. 网页端使用方法

1. 在右侧 `Username` 输入框中输入一个用户名，例如：

```text
apple_user_1
```

2. 点击 `Login`，页面会加载该用户的宠物状态。

3. 查看状态面板：

```text
Level：宠物等级
EXP：当前等级经验值
Food：剩余猫咪口粮数量
```

4. 点击 `Feed 喂食`：

```text
每次消耗 1 个口粮
每次增加 10 EXP
每满 100 EXP 升 1 级
```

如果没有口粮，页面会提示 `Not enough food rations`。

## 4. Apple Health 快捷指令绑定

本项目使用 iOS 快捷指令同步 Apple Health 步数，不使用 OAuth。

### 4.1 准备快捷指令

在 iPhone 上创建一个快捷指令，建议命名为：

```text
VitalPet 数据同步
```

快捷指令需要完成以下操作：

1. 读取 Apple Health 今日步数。
2. 询问或保存用户输入的 `username`。
3. 向 VitalPet 后端发送 POST 请求。

请求地址：

```text
https://你的域名/api/sync_health
```

本地测试时可以使用：

```text
http://你的电脑局域网 IP:5000/api/sync_health
```

请求方法：

```text
POST
```

请求头：

```text
Content-Type: application/json
```

请求体示例：

```json
{
  "username": "apple_user_1",
  "steps": 5200
}
```

### 4.2 设置网页二维码

将 iOS 快捷指令发布为 iCloud 分享链接后，打开：

```text
frontend/script.js
```

找到：

```js
const SHORTCUT_URL = "https://www.icloud.com/shortcuts/REPLACE_WITH_YOUR_VITALPET_SHORTCUT";
```

替换为你的真实快捷指令链接：

```js
const SHORTCUT_URL = "https://www.icloud.com/shortcuts/你的快捷指令ID";
```

刷新网页后，右侧二维码会自动更新。用户用 iPhone 扫描二维码即可添加快捷指令。

## 5. 游戏规则

步数兑换：

```text
每 1000 步 = 1 个猫咪口粮
向下取整
```

例如：

```text
5200 步 = 5 个口粮
999 步 = 0 个口粮
10000 步 = 10 个口粮
```

喂食规则：

```text
点击 Feed 喂食
消耗 1 个口粮
增加 10 EXP
```

升级规则：

```text
每 100 EXP 升 1 级
升级后 EXP 从剩余值继续计算
```

## 6. API 说明

### 6.1 同步健康数据

```http
POST /api/sync_health
```

请求体：

```json
{
  "username": "apple_user_1",
  "steps": 5200
}
```

成功返回：

```json
{
  "status": "success",
  "message": "Health data synced",
  "added_food": 5,
  "pet": {
    "username": "apple_user_1",
    "level": 1,
    "exp": 0,
    "food": 5,
    "total_steps": 5200
  }
}
```

### 6.2 获取宠物状态

```http
GET /api/pet_status?username=apple_user_1
```

成功返回：

```json
{
  "username": "apple_user_1",
  "level": 1,
  "exp": 0,
  "food": 5,
  "total_steps": 5200
}
```

### 6.3 喂食

```http
POST /api/feed
```

请求体：

```json
{
  "username": "apple_user_1"
}
```

成功返回：

```json
{
  "status": "success",
  "message": "+10 EXP",
  "pet": {
    "username": "apple_user_1",
    "level": 1,
    "exp": 10,
    "food": 4,
    "total_steps": 5200
  }
}
```

口粮不足时返回：

```json
{
  "status": "error",
  "message": "Not enough food rations",
  "pet": {
    "username": "apple_user_1",
    "level": 1,
    "exp": 10,
    "food": 0,
    "total_steps": 5200
  }
}
```

## 7. 常用测试命令

同步 5200 步：

```powershell
Invoke-RestMethod -Method Post `
  -Uri http://127.0.0.1:5000/api/sync_health `
  -ContentType 'application/json' `
  -Body '{"username":"apple_user_1","steps":5200}'
```

查看宠物状态：

```powershell
Invoke-RestMethod -Uri 'http://127.0.0.1:5000/api/pet_status?username=apple_user_1'
```

喂食一次：

```powershell
Invoke-RestMethod -Method Post `
  -Uri http://127.0.0.1:5000/api/feed `
  -ContentType 'application/json' `
  -Body '{"username":"apple_user_1"}'
```

## 8. 部署说明

### Render

推荐使用 Render 部署 Flask 后端。

构建命令：

```bash
pip install -r requirements.txt
```

启动命令：

```bash
gunicorn backend.app:app
```

项目已包含 `Procfile`：

```text
web: gunicorn backend.app:app
```

部署完成后，将快捷指令请求地址改为：

```text
https://你的 Render 域名/api/sync_health
```

### 自定义域名

绑定 `.me` 域名后，需要在快捷指令和网页二维码中使用正式域名：

```text
https://你的域名.me/api/sync_health
```

## 9. 注意事项

1. `username` 是用户唯一标识，网页端和 iOS 快捷指令中必须保持一致。
2. `steps` 表示本次同步新增的步数。当前后端会把传入步数累加到 `total_steps`。
3. 如果快捷指令每天上传的是“今日总步数”，一天内多次运行会重复累计。建议快捷指令每天只自动运行一次，或后续增加按日期去重逻辑。
4. SQLite 文件位于 `backend/vitalpet.db`。部署到无持久磁盘的平台时，数据库可能会在重启后丢失。
5. 上线后建议使用 HTTPS 域名，iOS 快捷指令请求会更稳定。

## 10. 故障排查

页面打不开：

```text
确认 Flask 服务是否已启动
确认访问地址是否为 http://127.0.0.1:5000/
确认 5000 端口没有被其他程序占用
```

登录后没有数据：

```text
确认 username 是否和快捷指令中的 username 一致
可以先调用 /api/sync_health 手动同步一组测试步数
```

二维码不是你的快捷指令：

```text
替换 frontend/script.js 中的 SHORTCUT_URL
刷新网页
```

喂食失败：

```text
确认 Food 数量是否大于 0
先同步至少 1000 步来获得口粮
```
